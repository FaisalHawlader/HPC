    ◆ choir_loop_once()

    int CHOIR_CLIENT choir_loop_once ( choir_socket_handler_t *  choir_handler)

    choir_loop_once is a non blocking function that check for received distribute, distribute_rel or notify_req to pass to the callback choir_loop_once can be set to blocking by settings a timeout with the choir_loop_set_timeout function On linux without timeout : the function return 0 and EAGAIN ("Resource temporarily unavailable") if no data has been received the function return -1, errno is not set by system if mw-server end the TCP stream

    On linux with timeout : the function return 0 and EAGAIN ("Resource temporarily unavailable") if no data has been received the function return -1, errno is not set by system if mw-server end the TCP stream

    On threadx without timeout : the function return 0 and EAGAIN ("No more processes") if no data has been received the function return -1 and ENOTCONN ("Socket is not connected") if mw-server end the TCP stream

    On threadx with timeout : the function return 0 and ETIMEDOUT ("Connection timed out") if no data has been received the function return -1 and ENOTCONN ("Socket is not connected") if mw-server end the TCP stream

    Parameters
    choir_handler
    Returns
    -1 on error 0 if nothing has been received > 0 otherwise
    Examples:
    /mw-c-api/binaries/src/mwcache.c, /mw-c-api/binaries/src/mwpubsub.c, /mw-c-api/examples/src/pubsub_example.c, and /mw-c-api/examples/src/subscribe_example.c.
