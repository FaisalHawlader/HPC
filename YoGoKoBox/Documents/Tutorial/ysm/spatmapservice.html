    Signal Phase and Time Service (SpatService) And Map Service

    The SpatService is in charge of :

    • sending and receiving SPATEM and MAPEM messages if configured with ETSI standard (European communication stack).
    • generating SPAT and MapData if configured with SAE standard (US comunication stack).

    Selection of European or US communication stack is done by selecting the correct standard reference and version in the service configuration file.

    It may be used by applications to implement the ETSI Traffic Light Maneuver (TLM) and Road and Lane Topology (RLT) services.

    Currently supported standard version for European communication stack :

    • ETSI TS 103 301 V1.1.1 (2016-11) with ISO/TS 19091:2016(E)
    • ETSI TS 103 301 V1.2.1 (2016-11) with ISO/TS 19091:2016(E)*
    • ETSI TS 103 301 V2.1.1 (2021-03) with ISO/TS 19091:2018(E)

    *Because of binary compatibility between ASN.1 module from version 1.2.1 and from version 2.1.1, v1.2.1 and v2.1.1 based messages are also compatible. Due to this, if the service is configured with version 1.2.1, version 2.1.1 will be used for communication. The managment of sent/received messages depending on the configured version is detailed in the following table :

    Sent message \ Receiver configuration v1.1.1 v1.2.1 v2.1.1
    v1.1.1 fully decoded not decoded not decoded
    v1.2.1 not decoded fully decoded fully decoded
    v2.1.1 without extension not decoded fully decoded fully decoded
    v2.1.1 with extension not decoded fully decoded fully decoded

    Currently supported standard version for US communication stack :

    • SAE J2735 2016

    Processing and I/O

    European communication stack

    On emission, SPATEM and MAPEM messages are generated and sent by the SpatService at regular time intervals. Messages are generated based on input or configuration provided to the service.

    On reception, SPATEM and MAPEM messages are passed to the upper layer, that is applications and services subscribed for reception of CAM messages.

    SpatService

    US communication stack

    In US mode, SpatService will only generate SPAT and MapData messages and then publish then on outspat and outmap message id. Communication with the network layer is done by the FrameService only.

    Generation of SPATEM and MAPEM messages (ETSI), or SPAT and MapData messages (SAE)

    SPATEM message is build from the itsPduHeader and a SPAT structure. MAPEM messages is build from the itsPduHeader and a MapData structure.

    SPAT and MapData structure in SPATEM and MAPEM messages or SPAT and MapData as messages are generated from configuration data provided by the user.

    Some configurations are get from MIBs:

    • The intersection identifier and region identifier
    • The list of signal groups for this intersection

    Some configuration are received from messages received on the msgid pubsub.configspat, under a JSON format. The SpatService supports 2 configuration messages:

    • MAP config.: Reference position and list of lanes of the intersection
    • SPAT config.: Signal group next states

    From these informations, the SPATEM and MAPEM messages are generated by the SpatService at regular intervals, as configured by emission.interval.

    MapData configuration:

    Field JSON type Description
    referencePosition object Reference position of the intersection
    referencePosition.latitude real Latitude of the intersection
    referencePosition.longitude real Longitude of the intersection
    lanes array List of lanes of this intersection
    lanes[i].id integer Lane ID
    lanes[i].ingress boolean Whether this lane enters the intersection
    lanes[i].egress boolean Whether this lane exits the intersection
    lanes[i].signalgroup integer Signal group associated to connections from this lane
    lanes[i].nodes array List of nodes of this lane
    lanes[i].nodes[j] string Node coordinates, delta from referencePosition in meters, consisting of delta latitude and delta longitude separated by semicolon

    SPAT configuration:

    Field JSON type Description
    referenceTime integer Time from which states are expressed
    signalGroup integer Signal group to configure
    states array List of states of this signal group
    states[i].state integer Signal state, as in SPATEM standard
    states[i].duration integer Duration of this state (in seconds)

    The SpatService currently supports basic configurations. More possibilities will be available in a future version.

    Emission of SPATEM and MAPEM messages (only in European mode)

    On emission, the SpatService subsscribes to messages on msgid pubsub.outspat for SPATEM messages and pubsub.outmap for MAPEM messages.

    If the securityparameters.securitycontrol configuration is enabled, SPATEM and MAPEM are first checked against security permissions (SSPs). Messages are then encoded into ASN.1 and sent on 2 itssockets, respectively socket.spat for SPATEM messages and socket.map for MAPEM messages.

    After being successfully sent, messages are republished using pubsub, on msgid pubsub.ownspat for SPATEM messages and pubsub.ownmap for MAPEM messages.

    Reception of SPATEM and MAPEM messages (only in European mode)

    SPATEM and MAPEM are received from 2 itssockets, respectively socket.spat for SPATEM messages and socket.map for MAPEM messages.

    On reception, the SpatService first decodes the ASN.1 structure of messages. Then, if the securityparameters.securitycontrol configuration is enabled, messages are checked against security permissions (SSPs). Finally, messages are passed to the upper layer using pubsub, respectively on msgid pubsub.inspat for SPATEM messages and pubsub.inmap for MAPEM messages.

    Sample configuration file

    config/40-spatservice.xml

    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
    <!--
    SpatService
    Provides support for ETSI SPATEM and MAPEM messaging
    -->
    <service start="true" instance="0">SpatService</service>
    <config>
    <!-- Parameters for the pubsub mechanism (reasonable defaults) -->
    <pubsub spatsenderid="71" spatreceiverid="72" spathandlerid="73" mapsenderid="81" mapreceiverid="82"
    inspat="24" outspat="34" ownspat="44" configspat="54" inmap="23" outmap="33" ownmap="43" />
    <!-- Parameters for the cache mechanism (reasonable defaults) -->
    <cache id="2"/>
    <!-- Parameters for DENM emission through LTE-V2X media
    * enable : Whether the LTE-V2X media should be use for DENM emission.
    * spatmsgtype : The string that identifies the ITS message type for LTE-V2X emission. This string can be used
    for example in the MQTT topic if MQTT protocol is used for the LTE-V2X emission.
    * mapmsgtype : The string that identifies the ITS message type for LTE-V2X emission. This string can be used
    for example in the MQTT topic if MQTT protocol is used for the LTE-V2X emission.
    -->
    <ltev2x enable="false" spatmsgtype="spat" mapmsgtype="map" />
    <!-- Parameters for the itssocket mechanism (reasonable defaults) -->
    <socket map="gn+btp://cch:2003" spat="gn+btp://cch:2004" />
    <!-- Standard reference and version to use -->
    <standard reference="ETSI_TS_103301" version="2.1.1" />
    <!-- <standard reference="ETSI_TS_103301" version="1.1.1" /> -->
    <!-- <standard reference="SAE_J2735" version="2016" /> -->
    <!-- Message emission parameters
    interval: base interval for message emission (ms)
    -->
    <emission interval="1000" />
    <!-- Sender Parameters
    updateheader: Allow senders to update header of messages
    -->
    <sender updateheader="false"/>
    <!-- List of signal groups -->
    <!-- number of signal groups -->
    <index size="0"/>
    <!-- IDs of signal groups (starting from 1) -->
    <!-- <signalgroup1 id="89" /> -->
    <!-- <signalgroup2 id="111" /> -->
    <!-- <signalgroup3 id="24" /> -->
    </config>
    </configuration>
