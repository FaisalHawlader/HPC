    Radio Management for V2X communications

    The RadioManagement service manages the radio interfaces for V2X communications.

    It manages the emission power and parameters according to the situation of the station, in order to implement radio management features (ETSI G5/CEN DSRC mitigation, etc).

    Processing and I/O

    The RadioManagement service gathers information from other services in the C-ITS communication stack and apply appropriate parameters to the radio interfaces accordingly.

    In the current version, the RadioManagement service only uses informations stored in the protectedzones cache, fed by the LdmService from CAM messages.

    RadioManagement

    Radio Management features

    In the current version, the RadioManagement service only supports the ETSI G5/CEN DSRC mitigation feature.

    ETSI G5/CEN DSRC mitigation

    The ETSI G5/CEN DSRC mitigation feature aims to protect toll systems from interference when a ETSI G5 enabled vehicle drives through or near a toll system on a motorway. To mitigate the risk of interferences, the vehicle shall reduce the transmit power on the ETSI G5 radios, and/or reduce the channel occupation.

    The RadioManagement service implements the following standard:

    • ETSI TS 102 792 V1.2.1 (2015-06)

    As per the ETSI TS 102 792 standard, the ITS station is to be considered as an ITS station conforming with default ITS radio parameters. In this current version, only the coexistence mode A is supported.

    Note: As required to operate under coexistence mode A, it is assumed that the OBU installation is equipped with an external antenna with at least a 1,5m cable.

    Considering the ETSI TS 102 792 standard, the RadioManagement service gathers protected zones as follows:

    • CAM ProtectedZone containers: Protected zones are received from CAM ProtectedZone containers and mitigation is applied accordingly.
    • Protected zone database (section 5.2.4): No permanent database is implemented
    • Short range protected zone radio detection (section 5.2.5): No radio detection is implemented

    When a CAM message is received with a ProtectedZones container, the protected zones found in the message are stored in a local cache, then the position of the vehicle is matched against the stored protected zones to identify whether a mitigation technique should be applied. Finally, in case the ETSI G5/CEN DSRC mitigation feature is enabled (using the mitigation.enabled configuration variable), the Radio Management service sets mitigation.active to true if the mitigation should apply, and applies parameters to the radio interfaces.

    Radio interface providers

    The RadioManagement service does not dependent on a specific radio interface and thus may apply parameters to several radio providers. The radio provider of each interface is to be specified in the configuration.

    Autotalks

    The autotalks provider implements the new behavior for controlling TX power on the Autotalks Craton 1 platform.

    The autotalks provider addresses a MIB SET request to the AutotalksServices service. It controls the keys v2x_radioctrl<id>.maxtxpower, where <id> is the identifier of the radio interface.

    The interfaces are identified by their interface number (e.g. 1 or 2).

    Atbridge

    The atbridge provider implements the legacy behavior for controlling TX power. It only applies on packets that are passing through the v2x0 or v2x1 interfaces.

    The atbridge providers sets the maximum TX power to the cache ID and object ID identified respectively by cache.propertiescacheid and cache.propertiesobjectid.

    The interfaces are identified by their network interface name (e.g. v2x0 or v2x1).

    Configuration and MIBs

    List of MIB entries

    Details of MIB entries:

    Section Field R/W Type Default value Description
    cache protectedzonecacheid RW integer 13 Cache ID of the Protected Zones cache
    cache propertiescacheid RW integer 30 Cache ID of the radio props cache
    cache propertiesobjectid RW integer 1 Object ID of the radio props cache
    radio1 provider RW string - Provider of the radio interface 1 (autotalks or atbridge)
    radio1 identifier RW string - Identifier of the radio interface 1 (provider-dependent)
    radio1 standardtxpower RW integer 23 Max TX power of the radio 1 under normal conditions (dBm)
    radio1 reducedtxpower RW integer 13 Max TX power of the radio 1 under reduced conditions (dBm)
    radio2 provider RW string - Provider of the radio interface 2 (autotalks or atbridge)
    radio2 identifier RW string - Identifier of the radio interface 2 (provider-dependent)
    radio2 standardtxpower RW integer 23 Max TX power of the radio 2 under normal conditions (dBm)
    radio2 reducedtxpower RW integer 13 Max TX power of the radio 2 under reduced conditions (dBm)
    mitigation enabled RW boolean true Whether ETSI G5/CEN DSRC mitigation should be enabled
    mitigation active R boolean - Whether ETSI G5/CEN DSRC mitigation is currently applied

    Sample configuration file

    config/15-radiomanagement.xml

    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
    <!--
    RadioManagement
    Manages radio parameters according to the situation of the C-ITS Station.
    -->
    <service start="true" instance="0">RadioManagement</service>
    <config>
    <!-- Parameters for the cache mechanism (reasonable defaults) -->
    <cache propertiescacheid="30" propertiesobjectid="1" protectedzonecacheid="13" />
    <!-- Parameters for the radio interfaces 1 or 2
    provider: Provider for the radio interface (autotalks or atbridge)
    identifier: Identifier of the radio interface (1 or 2 for autotalks,
    v2x0 or v2x1 for atbridge)
    standardtxpower: TX power in normal (non DSRC-mitigated) mode
    reducedtxpower: TX power in reduced (DSRC-mitigated) mode
    -->
    <radio1 provider="autotalks"
    identifier="1"
    standardtxpower="23"
    reducedtxpower="13" />
    <radio2 provider="autotalks"
    identifier="2"
    standardtxpower="20"
    reducedtxpower="10" />
    <!-- Mitigation parameters
    enabled: Whether the mitigation mechanism should be enabled
    -->
    <mitigation enabled="false" />
    </config>
    </configuration>

    Usage examples

    Check whether the radio is under mitigation

    $ mwconfig mib get -t RadioManagement mitigation.active

    Output:

    mitigation.active = true
