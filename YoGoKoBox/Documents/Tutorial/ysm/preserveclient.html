    Preserve Client

    Preserve Client connects the Y-SMART Middleware to the PRESERVE V2X Security Subsystem (VSS) through the convergence layer of the later.

    This service is an implementation of the interfaces between C-ITS Facilities layer and C-ITS Security entity. As such it is currently mainly responsible of negotiating Authorization Ticket change along with the associated station ID, after a network protocol agreement with the PRESERVE VSS.

    The overall functioning of the security architecture in the Y-SMART Middleware is described on page Station Identity manager. This present page only details configuration variables of the Preserve Client service.

    Configuration and MIBs

    List of MIB entries

    Details of MIB entries:

    Section Field Type Default value Description
    pubsub publish_id integer 170 Application ID used to identify
    pubsub cache_id integer 3 Cache ID used to set station parameters
    pubsub cache_obj_id integer 1 Cache Object ID used to set station parameters
    server address string 127.0.0.1 Hostname/IP of the PRESERVE VSS Convergence Layer
    server port integer 8367 Port of the PRESERVE VSS Convergence Layer
    server modechangeport integer 28001 UDP Connectless port, used to change the mode of the PRESERVE VSS
    modes count integer - Service number of available mode
    mode<n> name string - Service available modes
    mode current string - Service current active mode

    Sample configuration file

    config/30-preserveclient.xml

    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
    <!--
    PreserveClient
    Connects to the PRESERVE V2X Security Subsystem (VSS)
    -->
    <service start="false" instance="0">PreserveClient</service>
    <config>
    <!-- Parameters for the pubsub and cache mechanism (reasonable defaults) -->
    <pubsub publish_id="170" cache_id="3" cache_obj_id="1" />
    <!--
    PRESERVE V2X Security Subsystem (VSS)
    address: address of the PRESERVE VSS Convergence Layer
    port: port of the PRESERVE VSS Convergence Layer
    modechangeport: UDP Connectless port, used to change the mode of the station
    -->
    <server address="127.0.0.1"
    port="8367"
    modechangeport="28001"/>
    <!-- security mode available
    number of available modes -->
    <modes count="2" />
    <!-- available modes -->
    <mode1 name="user" />
    <mode2 name="operator" />
    <!-- current security mode -->
    <mode current="user" />
    </config>
    </configuration>

    Usage examples

    Get the current mode of the Preserve Client

    $ mwconfig mib get -t PreserveClient mode.current

    Output:

    $ mode.current = "user"

    Change the current mode of the Preserve Client

    $ mwconfig mib set -t PreserveClient "mode.current=\"operator\""

    Output:

    None

    Retrieve the full information of the Preserve Client

    $ mwconfig mib get-all -t PreserveClient

    Output:

    pubsub.publish_id = 170
    pubsub.cache_id = 3
    pubsub.cache_obj_id = 1
    server.address = "127.0.0.1"
    server.port = 8367
    server.modechangeport = 28001
    mode.current = "operator"
    modes.count = 2
    mode1.name = "user"
    mode2.name = "operator"
