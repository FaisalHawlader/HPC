YoGoKo Smart Middleware

    Pkiservice

    Overview

    Y-Smart Middleware is extended by complementary YoGoKo components. PKIService is one of those. It implements the trust and privacy management for Cooperative Intelligent Transport System (C-ITS) communications.

    Basically, the PKIService manages the certificates of a C-ITS Station using a Public Key Infrastructure (PKI). It allows the generation of Long-Term Certificates (LTC) from Enrolment Authority (EA), the ability to request Pseudonym Certificates from Authorization Authority (AA) and transmits them to the Geonetworking Security Stack.

    YoGoKo PKIService and itsnet / libgn implement the so-called ETSI PKI v1 standards, and more specifically the following standards:

    • ETSI TS 102 940 – v1.1.1 : ITS security
    • ETSI TS 102 941 – v1.1.1 : ETSI trust model
    • ETSI TS 102 731 – v1.1.1 : Security services
    • ETSI TS 103 097 – v1.2.1 : ETSI messages and certificates formats

    C-ITS Security implementation

    The PKIService enables the implementation of the C-ITS Security in the C-ITS Station, along with itsnet or libgn. Security operations are distributed among these software components:

    The PKIService has the responsibility of communicating with a PKI in order to download certificates and trust or revocation lists. It manages pseudonyms and their rotation or their renewal from the PKI. It then regularly loads new parameters (pseudonyms and trust store) to the Geonetworking security stack.

    YoGoKo itsnet / libgn is in charged of per-packet security operations, that is: signing outgoing packets with the current pseudonym and verifying incoming packets.

    PKIService configuration files

    Configuration folder

    The configuration folder of the PKIService is generally located at /etc/pkiservice/.

    It contains the following files:

    • pkiconf.xml - Main configuration file
    • log4j.properties - Logging configuration
    • rca.cert - Certificate of the Root Certificate Authority

    Main configuration file

    The pkiconf.xml contains configuration parameters for the PKIService. This configuration file is formatted in XML.

    The configuration is surrounded by a **<config>** tag. Then, configuration sections are located in the following tags:

    Setup and testing

    In order to have a functional PKI setup on a C-ITS Station, the following steps must be carried out :

    • Configure the PKIService and other services impacted by C-ITS Security
    • Generate the station Technical Key
    • Register the station with the PKI Provider
    • Testing the security setup

    Note: The PKIService is a PKI client, it allows to connect to a compatible PKI provider. The choice and contractualisation with a PKI provider is under the responsibility of the user.

    Configuration

    The PKIService configuration must be modified according to the expected setup (e.g. PKI Provider, pools, SSPs, etc).

    Additionally, security settings must be set appropriately in security-related services:

    • Itsnet
    • Station Identity manager
    • YoGoKo Security Client

    Technical Key Generation

    The registration of a Station to a PKI Provider requires the generation and the provision of a Technical Key (TPK) to bootstrap security.

    The procedure for generating the technical key depends on the security setup (OpenSSL or HSM). Thus, it is platform-specific.

    YoGoKo YBOX-VEHI-1603/YBOX-ROAD-1602 platforms

    The following procedure describes the generation of the TPK on a YBOX-VEHI-1603 / YBOX-ROAD-1602 platform and derivatives.

    On these platforms, private keys used for C-ITS Security are stored in a HSM (provided by the Autotalks hardware environment). The HSM master key is stored on a secured memory on the Gateworks board, it is initialized and unlocked by a program named itsnvmunlockd which comes along with itsnet.

    $ mkdir -p /var/lib/pkiservice/certs/
    # Stop services that may try to use the HSM
    $ systemctl stop pkiservice
    $ systemctl stop itsnvmunlockd
    # Reset the Autotalks NVM, Generate the technical key
    $ itsnvmunlockd --newkeys --resethsm > /var/lib/pkiservice/technicalkey
    # Restart services that were stopped
    $ systemctl start pkiservice
    $ systemctl start itsnvmunlockd

    Registration of the C-ITS Station to the PKI Provider

    The registration of a C-ITS Station to a PKI Provider is provider-specific. YoGoKo Support team may assist users in this task.
