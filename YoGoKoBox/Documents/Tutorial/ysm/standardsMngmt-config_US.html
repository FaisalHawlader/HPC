YoGoKo Smart Middleware

    US communication stack configuration

    This page describes how configuration files of YoGoKo communication stack services must be filled to support IEEE / SAE communication stack.

    Standards supported in this configuration are :

    • WAVE : IEEE P1609.3v3
    • BSM : SAE J2735 2016
    • FRAME : SAE J2735 2016
    • SPAT/MapData : SAE J2735 2016

    Activate EPD protocol discrimination

    10-autotalks.xml

    <config>
    ...
    <channel wlanepdenabled=1/>
    <v2x_link1 frame_format="epd" frame_autodetect="false"/>
    <v2x_link2 frame_format="epd" frame_autodetect="false"/>
    ...
    </config>

    Select choir itsnet socket for communication between the middleware and the network layer

    10-v2xservice.xml

    <config>
    ...
    <choiritsnet1 channel="cch" appid="83" sendmsgid="12" recvmsgid="11" />
    ...
    </config>

    Configure itsnet to use WAVE network protocol

    20-itsnet.xml

    <config>
    ...
    <protocol type="WAVE" />
    ...
    </config>

    Activate messaging services and select standard to use

    40-bsservice.xml

    <service start="true" instance="0">BsService</service>
    <config>
    ...
    <standard reference="SAE_J2735" version="2016" />
    ...
    </config>

    40-frameservice.xml

    <service start="true" instance="0">FrameService</service>
    <config>
    ...
    <standard reference="SAE_J2735" version="2016" />
    ...
    </config>

    40-spatservice.xml

    <service start="true" instance="0">SpatService</service>
    <config>
    ...
    <standard reference="SAE_J2735" version="2016" />
    ...
    </config>
