YoGoKo Smart Middleware

    Standards Management

    Overview

    From version 2.7, YoGoKo’s communication stack can be configured to support several ITS standards :

    • v1 (2016) and v2 (2019) ETSI standards for european ITS communication stack
    • IEEE / SAE standards for US ITS communication stack

    The YoGoKo's middleware has been designed to support all these standards reference and version in a same build of the software.

    Upper layer services and API access (LDM, SpatMap service, itsnet) are designed to be totally abstract of the used standards.

    Configuration

    Configuration of standards to use is done by configuration.

    Each ITS communication stack involves specific standard selection for several layers :

    layer EU communication stack US communication stack
    Network ETSI GeoNetworking IEEE WAVE
    Facility ETSI / ISO SAE

    Access configuration

    Access layer configuration is done in the autotalks configuration file.

    Protocol discrimination format can be set to LPD (LLC Protocol Discrimination) for european communication stack or EPD (Ethertype Protocol Discrimination) for US communication stack. This setting is done through the v2x_link1.frame_format and v2x_link2.frame_format variables and can be set to lpd or epd.

    Network configuration

    Network protocol support is done by Itsnet.

    Itsnet can be configured to support GeoNetworking or WAVE network protocol.

    Facility configuration

    YoGoKo’s communication stack handles several facilities communication services.

    Some are specific to the ETSI European communication stack, other are specific to the US communication stack, and others are common to both.

    European communication services :

    • Cooperative Awareness Message Service (CaService)
    • Decentralized Environmental Notification Basic Service (DenService)
    • Infrastructure to Vehicle Information Service (IviService)

    US communication services :

    • Basic Safety Message Service (BsService)
    • DSRC Frame Message Service (FrameService)

    Common communication service :

    • Signal Phase and Time Service (SpatService) And Map Service

    Activation and configuration of each service must be done in its specific configuration file.

    The standard to use in each service must be indicated with the folowing configuration variables :

    • standard.reference : reference name of the standard
    • standard.version : version number of the standard referenced below

    Futhermore, current standard used in each service can be view in the Metadata of messages published accessible through the API (in and own message id of each message type).

    Security configuration

    Only ETSI security standards are supported.

    Multi-standard security support is not yet implemented. It is planned for a future release.

    Supported security standards are listed in the ETSI security architecture section on page Overview.
