YoGoKo Smart Middleware

    YoGoKo Security Client

    YoGoKo Security Client connects YoGoKo Middleware to other YoGoKo components involved in implementation of C-ITS security (YoGoKo PKI Service and Itsnet).

    The overall functioning of the security architecture in the middleware is described on page Station Identity manager. This present page only details configuration variables of the Security Client service.

    Configuration and MIBs

    List of MIB entries

    Details of MIB entries:

    Section Field R/W Type Default value Description
    choir applicationid RW integer 11 App ID to subscribe with
    choir securitycacheid RW integer 3 Cache ID of Security parameters cache
    choir identitycacheid RW integer 0 Cache ID of Station parameters cache
    choir securitymsgid RW integer 170 Message ID to receive security lock messages from the PKI Service
    choir pkiservicemsgid RW integer 193 Message ID to publish ID change lock requests to the PKI Service

    Sample configuration file

    config/20-securityclient.xml

    <?xml version="1.0" encoding="utf-8"?>
    <configuration>
    <service start="true" instance="0">SecurityClient</service>
    <config>
    <!-- Message Id for choir protocol
    Note: The securitycacheid entry must not be set to the same value as the
    choir.identitycacheid MIB entry from the IdentityManager service. Such a
    situation would lead to an interlocking. -->
    <choir applicationid="11" securitycacheid="3" securitymsgid="170" />
    </config>
    </configuration>
