YoGoKo Smart Middleware

    Websocket API

    Overview

    General description

    This WebSocket API can be use to access differents internal mechanism of middleware, such as publish/subscribe, cache operations or config.

    API is accessible through ip of YoGoKo Y-Smart Middleware and port 8080. Current browser's implementations of websocket does not handle the Websocket ping mechanism. In order to keep-alive the websocket connection and detect connection losses, the Websocket API implements a ping mechanism. When the user sends a '__ping__' through the websocket connection, the websocket server replies with a '__pong__'. This feature may be enabled and controlled in the choirconf.xml configuration file.

    Websocket API Methods

    PubSub

    Publish

    Publishes a message for delivery to all subscribers.

    Endpoints:

    • /pubsub/pub/{msg-id}
    • /pubsub/pub/{msg-id}/{app-id}
    • /pubsub/pub/{msg-id}/{app-id}/{format}
    Parameters Description Default value
    msg-id The Message id to publish to.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    format Format use to publish messages. Possible values : 's0' = ANY, 's1' = GENERIC, 's2' = ASN1_UPER, 's7' = JSON, 'b' = BINARY 's7'

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user may send messages. Each message will be published on the message id as specified in parameters.

    Subscribe

    Subscribes to a message id. All messages published to this message id will then be delivered.

    Endpoints:

    • /pubsub/sub/{msg-id}
    • /pubsub/sub/{msg-id}/{app-id}
    • /pubsub/sub/{msg-id}/{app-id}/{format}
    Parameters Description Default value
    msg-id The Message id to subscribe to.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    format Format use to subscribe messages. Possible values : 's0' = ANY, 's1' = GENERIC, 's2' = ASN1_UPER, 's7' = JSON, 'b' = BINARY 's7'

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user may receive messages. Each message published on the message id specified in parameters will be received.

    Request

    Publishes a message for reliable delivery to all subscribers, then wait for their response.

    Endpoints:

    • /pubsub/req/{msg-id}
    • /pubsub/req/{msg-id}/{app-id}
    • /pubsub/req/{msg-id}/{app-id}/{format}
    Parameters Description Default value
    msg-id The Message id to request to.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    format Format use to request messages. Possible values : 's0' = ANY, 's1' = GENERIC, 's2' = ASN1_UPER, 's7' = JSON, 'b' = BINARY 's7'

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user may send messages and receive a response if message has been delivered.

    Cache

    Get

    Retrieve an object from the cache.

    Endpoints:

    • /cache/get/{cache-id}/{object-id}
    • /cache/get/{cache-id}/{object-id}/{app-id}
    Parameters Description Default value
    cache-id Id that identify kind of cache.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    object-id Id that identify an object.

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user will receive the object as specified in parameters in JSON format.

    Set

    Push modifications to the cache.

    Endpoints:

    • /cache/set/{cache-id}/{object-id}
    • /cache/set/{cache-id}/{object-id}/{app-id}
    Parameters Description Default value
    cache-id Id that identify kind of cache.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    object-id Id that identify an object.

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user may send a message with the update in JSON format.

    Remove

    Remove an object from the cache.

    Endpoints:

    • /cache/remove/{cache_id}/{object-id}
    • /cache/remove/{cache-id}/{object-id}/{app-id}
    Parameters Description Default value
    cache-id Id that identify kind of cache.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    object-id Id that identify an object.

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the object will be removed from the cache if it exists.

    List

    List objects of a cache.

    Endpoints:

    • /cache/list/{cache-id}/
    • /cache/list/{cache-id}/{app-id}
    Parameters Description Default value
    cache-id Id that identify kind of cache.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user will receive the list of object-id of all object store in cache in JSON format.

    Register

    Register to modifications performed on an object.

    Endpoints:

    • /cache/register/{cache-id}/{object-id}
    • /cache/register/{cache-id}/{object-id}/{app-id}
    Parameters Description Default value
    cache-id Id that identify kind of cache.
    app-id The Application id to register with. Can be generate if not provided or equal to 0 0
    object-id Id that identify an object. 0 is only valid to match all objects.

    If app-id = 0, a random application ID will be generated, based on min/max _dynamic_id. Once connected to the websocket endpoint, the user will receive messages in JSON format everytime object is updated.

    Config

    Configure middleware service.

    Endpoint:

    • /config
    Command Description Message
    List List Middleware's Services {"command":"list-services"}
    Dump Get all services status and current mibs {"command":"dump"}
    Control status Know the status of service {"command":"config-status", "service-name":"<name_service>" ,"offline" : "true"}
    Control start Start a service {"command":"config-start", "service-name":"<name_service>" ,"offline" : "true"}
    Control stop Stop a service {"command":"config-stop", "service-name":"<name_service>" ,"offline" : "true"}
    Control restart Restart a service {"command":"config-restart", "service-name":"<name_service>" ,"offline" : "true"}
    mib get-all Get all mib of a service {"command":"mib-getall", "service-name":"<name_service>" ,"offline" : "true"}
    mib get Get specified mib entry {"command":"mib-get", "service-name":"<name_service>" ,"offline" : "true", "keys": ['<name_variable>']}
    mib set Set specified configuration in mib {"command":"mib-set", "service-name":"<name_service>" ,"offline" : "true", "mib": {"<name_variable>": <new_value>}}
    mib reset Reset mib {"command":"mib-reset", "service-name":"<name_service>" ,"offline" : "true"}

    Once connected to the websocket endpoint, the user will can send its command to manage YoGoKo middleware's services.

    Usage:

    • {"lat":48.8671769,"lon":2.7838265} -> 'ws://127.0.0.1:8080/pubsub/pub/201/65123/s7' -> Allow you to publish position fix in JSON format.
    • 'ws://127.0.0.1:8080/pubsub/sub/32/65124/b' -> Allow you to subscribe to messages CAM received in binary format.
    • { (...) } -> 'ws://127.0.0.1:8080/pubsub/req/25/4545' -> Allow you to publish message in JSON format to trigger denm event.
    • 'ws://127.0.0.1:8080/cache/get/11/281475632005121' -> Allow you to get object 281475632005121 in LDM events cache.
    • {"causecode": 11} -> 'ws://127.0.0.1:8080/cache/set/11/281475632005121' -> Allow you to change causecode of object 281475632005121 in denm cache of LDM.
    • 'ws://127.0.0.1:8080/cache/remove/11/281475632005121' -> Allow you to remove object 281475632005121 from LDM events cache.
    • 'ws://127.0.0.1:8080/cache/list/10/' -> Allow you to get the list of stations present in LDM stations cache.
    • 'ws://127.0.0.1:8080/cache/register/10/0' -> Allow you to receive every update of every object in LDM stations cache.
    • {"command":"list-services"} -> 'ws://127.0.0.1:8080/config' -> Allow you to see every middleware's services and their status.
