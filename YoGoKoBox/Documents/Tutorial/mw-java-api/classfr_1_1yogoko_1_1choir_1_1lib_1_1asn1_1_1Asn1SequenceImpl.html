    ◆ compareValue()

    boolean fr.yogoko.choir.lib.asn1.Asn1SequenceImpl.compareValue ( ASN1Component  obj)

    Compare the value represented by an ASN1Component with another ASN1Component. Note: If combined with compareDefinition(), it may be used to ensure that both components will generate the same encoding.

    Parameters
    objASN1Component object to compare with
    Returns
    Return whether both objects represent the same value

    We only compare the components that are to be included (not optional nor extended, or with option bit or extended bit set to 1), since other components are not going to be encoded and thus are not meaningfull at this present time.

    The way the component is included may be different in both compared sequences (e.g. one may be optional but option is set while the other is not optional). But as soon as the component is included in one sequence, it must be included in the compared one as well.

    Note that incrementing independently i and j makes it necessarily to ensure that we compare the same field.

    Two modes exist for extension marks :

    • One extension mark at the end of sequence, extended components are appended at the end of the sequence
    • Two extension marks followed by non-extended components, extended components are appended between these two extension marks, then followed by non-extended components For now, we only support the first case
