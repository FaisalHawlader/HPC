    fr.yogoko.choir.internals.ipc.Client Class Reference

    Detailed Description

    Client interface of the middleware Choir. This is the entry point of ITS-S facilities provided by this middleware. All the services are actually integrated in the Choir-server, thus the Choir-client just works as API for user applications. For this reason, to provide ITS-S facilities, this client must always be connected to the server.

    The server may be either in localhost or remote host. In any case, Client must connect to a server in the same ITS-S. (Conceptually, each client and server belongs to one ITS-S, and only one server must be in each ITS-S. Whereas there may be multiple clients in a single ITS-S.)

    ChoirMessage fr.yogoko.choir.internals.ipc.Client.sendControlMessage ( ChoirMessage  cm) throws NotYetConnectedException, IOException

    Send a control message to choir-server. Depending on the commandId, this method may block until the server sends back a reply.

    In case of the following commands, this method blocks at most

    TIMEOUT_MS

    milliseseconds: Connection request Message reception request from ItsSocket ItsSocket allocation & open request Message subscription request

    Parameters
    cmOutgoing control message
    Returns
    Replied control message, or
    null
    if there is no reply to the request
    Exceptions
    NotYetConnectedExceptionIf the client does not establish the connection to the server
    IOExceptionIf a network error occurred
